# CellRanger GEM-X Flex 4-plex Pipeline

## Description
A bash pipeline for running `cellranger multi` on GEM-X Flex Gene Expression data (4-plex, probe-based).  
Handles config generation, demultiplexing, QC summary, and outputs ready-to-load Seurat code.

---

## How GEM-X Flex Multiplexing Works

Unlike CMO or Hashtag-based multiplexing, GEM-X Flex uses **probe-based barcoding**:

- Each sample is hybridised with a unique **Probe Barcode (BC001–BC004)** at the molecular biology stage — *before* the samples are pooled
- The probe barcode travels with every RNA molecule through GEM encapsulation
- All 4 samples are pooled into **one GEM well → one pair of FASTQ files**
- There is **no separate multiplexing-capture library** — only Gene Expression FASTQs
- CellRanger demultiplexes purely from the probe barcode sequence embedded in reads
- A read is assigned to a sample only if it has valid barcodes in **both** the 10x GEM barcode and the probe barcode

> In simple terms: each sample is given a unique molecular tag before pooling, and CellRanger reads those tags to split everything back into individual samples.

Because of this, the correct pipeline is `cellranger multi` with a Flex-specific config — **not** `cellranger count`.

## Requirements

- CellRanger >= 7.0 (>= 10.0 recommended for GEM-X Flex v2)
- GRCh38 reference genome
- Probe set CSV (bundled inside your CellRanger installation)

---

## Key Configuration Variables

Before running, edit these variables in the script:

**`TRANSCRIPTOME`** — path to your GRCh38 CellRanger reference directory.

**`PROBE_SET`** — path to the probe set CSV inside your CellRanger installation.  
For Flex v2 use: `Chromium_Human_Transcriptome_Probe_Set_v2.0_GRCh38-2024-A.csv`  
For Flex v1 use: `Chromium_Human_Transcriptome_Probe_Set_v1.1.0_GRCh38-2024-A.csv`  
Find it under: `cellranger-X.Y.Z/probe_sets/`  
See: https://www.10xgenomics.com/support/software/cell-ranger/latest/analysis/running-pipelines/cr-flexv2-multi

**`FASTQ_DIR`** — directory containing `SO_14534_R1.fastq.gz` and `SO_14534_R2.fastq.gz`.

**`FASTQ_ID`** — the FASTQ filename prefix, currently set to `SO_14534`.

**`SAMPLE_TO_BC`** — mapping of sample names to probe barcode IDs.  
This **must match Table 2 of your NGS Library Report exactly**.  
If the sequencing facility swapped BC002 and BC003 during library prep, your demultiplexing will be wrong.

## Config Generated by the Script

```
[gene-expression]
reference,/path/to/GRCh38
probe-set,/path/to/probe_set.csv
create-bam,false

[libraries]
fastq_id,fastqs,feature_types
SO_14534,/path/to/fastqs,Gene Expression

[samples]
sample_id,probe_barcode_ids
SC001,BC001
SC002,BC002
SC003,BC003
SC004,BC004
```

## Usage

```bash
chmod +x cellranger_flex4plex_pipeline.sh

# Normal run
./cellranger_flex4plex_pipeline.sh

# Dry run (prints commands without executing)
./cellranger_flex4plex_pipeline.sh --dry-run

# Override paths at runtime
./cellranger_flex4plex_pipeline.sh \
  --transcriptome /path/to/ref \
  --probe-set /path/to/probe_set.csv \
  --fastq-dir /path/to/fastqs \
  --cores 16 --mem 128
```

## Output Structure

```
cellranger_out/
└── plaque_flex4plex/
    └── outs/
        ├── per_sample_outs/
        │   ├── SC001/
        │   ├── SC002/
        │   ├── SC003/
        │   └── SC004/
        └── multiplexing_analysis/
            ├── cells_per_tag.json
            └── frp_gem_barcode_overlap.csv
```

Each sample folder under `per_sample_outs/` contains its own filtered feature-barcode matrix, ready for loading into Seurat.  
The `frp_gem_barcode_overlap.csv` file is useful for assessing multiplet rates across samples.
